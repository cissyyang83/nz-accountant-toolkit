# Contributing to NZ Accountant Toolkit

Thank you for considering a contribution.

NZ Accountant Toolkit is intended to be practical, accurate, and readable for accountants and small business advisors. Contributions should improve real engagement workflows, not only technical structure.

## Who Can Contribute

Contributions are welcome from:

- Chartered accountants and CPA practitioners.
- Tax agents.
- Bookkeepers.
- Business advisors.
- Software developers working with accountants.
- Students and educators.

## Contribution Principles

- Use professional English.
- Explain accounting assumptions clearly.
- Avoid unsupported tax conclusions.
- Include reviewer prompts where judgement is required.
- Prefer New Zealand examples and terminology.
- Keep templates practical for small and medium practices.
- Do not include confidential client data.

## Suggested Contribution Types

- Template specifications.
- Checklist improvements.
- Example calculations.
- Plain-English tax explanations.
- Xero workflow notes.
- Python calculator prototypes.
- Test cases.
- Documentation corrections.

## Accounting and Tax Review

Before submitting content, check whether it:

- Identifies the relevant period and tax year.
- Separates facts, assumptions, and conclusions.
- Notes when Inland Revenue guidance or legislation should be checked.
- Includes reviewer sign-off points.
- Avoids presenting uncertain areas as settled law.

## Pull Request Checklist

Before opening a pull request:

- Read the relevant existing files.
- Keep changes focused.
- Add or update examples where useful.
- Add a brief explanation of the change.
- Confirm that no confidential client information is included.
- Run tests when changing Python code.

## Style Guide

- Write for accountants first.
- Use headings and tables where they help review.
- Define abbreviations such as FIF, FDR, CV, and GST when first used in a file.
- Use New Zealand spelling.
- Avoid unnecessary software jargon.

## Code Contributions

Python calculator contributions should:

- Keep calculation functions small and reviewable.
- Include tests for realistic New Zealand examples.
- Return review notes where professional judgement is required.
- Avoid deciding tax positions automatically.
- Prefer standard library code unless a dependency is clearly justified.

Run tests with:

```bash
pytest
```

## Disclaimer for Contributions

By contributing, you agree that your contribution may be used under the project licence and that it does not contain confidential, proprietary, or client-identifiable information.
