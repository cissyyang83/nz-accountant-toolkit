# Governance

NZ Accountant Toolkit uses a lightweight maintainer-led governance model.

## Decision Making

The primary maintainer makes final decisions on:

- Repository structure.
- Release timing.
- Accepted templates and examples.
- Project roadmap.
- Contributor roles.

Where a decision involves New Zealand tax or accounting interpretation, maintainers should seek review from a suitably experienced practitioner before presenting the content as part of a stable release.

## Project Values

- Practical usefulness for accountants.
- Plain-English documentation.
- Respect for professional judgement.
- No confidential client information.
- Transparent assumptions.
- Incremental improvement.

## Contribution Review

Pull requests are reviewed for:

- Practical accounting usefulness.
- New Zealand terminology.
- Clear assumptions.
- Confidentiality risk.
- Consistency with existing templates.
- Test coverage where code is changed.

## Releases

Releases should include:

- Version number.
- Summary of changes.
- New or changed templates.
- Known limitations.
- Any tax-sensitive areas needing practitioner review.

## Conflicts

If contributors disagree, maintainers should:

1. Restate the issue.
2. Separate facts, assumptions, and preferences.
3. Invite evidence or references.
4. Choose the option that best protects users and project quality.
