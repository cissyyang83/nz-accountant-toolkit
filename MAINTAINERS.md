# Maintainers

NZ Accountant Toolkit is maintained by the project owner and future community contributors.

## Current Maintainer

| Name | Role | Responsibilities |
| --- | --- | --- |
| Project owner | Primary maintainer | Roadmap, issue triage, releases, documentation quality, accounting review coordination |

## Maintainer Responsibilities

Maintainers are responsible for:

- Reviewing issues and pull requests.
- Protecting client confidentiality.
- Keeping documentation practical for New Zealand accountants.
- Coordinating accounting and tax review before releases.
- Publishing release notes.
- Maintaining project health files.

## Adding Maintainers

New maintainers may be added when they have:

- Made repeated useful contributions.
- Shown care with accounting assumptions and confidentiality.
- Helped review issues or pull requests.
- Demonstrated good judgement in New Zealand accounting or software workflows.

## Response Targets

These are targets, not guarantees:

| Activity | Target |
| --- | --- |
| Acknowledge new issues | Within 14 days |
| Review pull requests | Within 21 days |
| Publish release notes | For every tagged release |
| Review tax-sensitive wording | Before each release |
