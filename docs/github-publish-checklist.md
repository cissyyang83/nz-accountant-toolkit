# GitHub Publish Checklist

Use this checklist before applying to Codex for Open Source.

## Repository Setup

- Create a public GitHub repository named `NZ Accountant Toolkit` or `nz-accountant-toolkit`.
- Push the local repository to GitHub.
- Replace placeholder GitHub URLs in `pyproject.toml` and `.github/ISSUE_TEMPLATE/config.yml`.
- Replace `@cissyyang83` in `.github/CODEOWNERS`.
- Confirm the default branch is `main`.
- Confirm GitHub Actions is enabled.

## First Release

- Create tag `v0.1.0`.
- Publish release notes from `docs/release-v0.1.md`.
- Confirm `CHANGELOG.md` matches the release.

## Issues and Discussions

- Create starter issues from `docs/github-issues.md`.
- Create discussion categories and starter topics from `docs/github-discussions.md`.
- Label issues using the labels listed in `docs/maintenance-plan.md`.

## Application Evidence

- Add screenshots or workbook preview images when available.
- Add at least one real Excel workbook once ready.
- Invite accountants or bookkeepers to comment on the project.
- Triage at least a few public issues.
- Record maintainer activity through commits, issue comments, and releases.

## Codex for Open Source Application

- Confirm the GitHub profile is public.
- Confirm the repository is public.
- Prepare a concise explanation of ecosystem importance.
- Explain how Codex or API credits would support issue triage, PR review, release workflows, documentation review, and calculator development.
