# Release Checklist

Use this checklist before publishing a GitHub release.

## Preparation

- Version number selected.
- Changelog updated.
- Release notes drafted.
- Tests run successfully.
- Documentation links checked.
- Examples reviewed for confidentiality.

## Accounting Review

- Tax-sensitive wording reviewed.
- Assumptions clearly separated from conclusions.
- Disclaimers included where needed.
- Reviewer prompts included.
- New Zealand terminology checked.

## Code Review

- Calculator changes have tests.
- CLI examples still work.
- No credentials or private data included.
- GitHub Actions workflow passes.

## Publication

- Git tag created.
- GitHub release created.
- Known limitations included.
- Next roadmap items identified.
